1
สร้าง Working Revision
  • กดสร้างจากเอกสาร Published ที่ต้องการแก้ไข
  • ระบบสร้าง record ใน document_revision_drafts
  • Rev. ถัดไปถูกกำหนดอัตโนมัติ (เช่น Rev.06 → Rev.07)
  • ถ้ามี draft ค้างอยู่ → ระบบ return draft เดิม ไม่สร้างซ้ำ

2
กรอก Metadata
  • ชื่อเอกสาร, แผนก
  • วันที่แก้ไข / วันที่ทบทวน / วันที่บังคับใช้
  • ผู้จัดทำ / ผู้รับรอง / ผู้อนุมัติ
⚡ Side Effect ทันที
  • Update document_revision_drafts
  • Patch header Word/Excel ใน R2 อัตโนมัติ
DCC Admin
3
อัปโหลด Word / Excel (ไฟล์ต้นฉบับ)
  • รองรับ .docx / .xlsx
  • ระบบ patch header อัตโนมัติ: วันที่, รหัสเอกสาร, ชื่อ
  • บันทึก path ใน word_url ของ draft
  • จำเป็น สำหรับ QP/WI ก่อนส่ง Review
DCC Admin
4
อัปโหลด PDF ทางการ (ไฟล์เนื้อหา)
  • QP/WI: อัปโหลด PDF เนื้อหา → source_pdf_url
  • Form/อื่นๆ: อัปโหลดไฟล์ทางการ (PDF/Word/Excel) → file_url
  • จำเป็น ก่อนส่งเข้า Review ทุกประเภท
DCC เท่านั้น Admin

  • ระบบตรวจสอบเงื่อนไขก่อนอนุญาตเปลี่ยนสถานะ
